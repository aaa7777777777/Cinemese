# EVENTS.md — event subsystem spec

## What an event is

An event is anything that happened to the character
that is worth remembering for longer than one conversation.
It is not the conversation itself.
It is what the conversation left behind.

Events have two sources:
1. Generated by event_engine on schedule or by story trigger
2. Injected by planning.py based on user settings or another agent's collision

---

## Event schema

```yaml
id: str
ts: ISO timestamp
type: quiet_moment | interaction | discovery | conflict | internal_shift
source: "scheduled" | "story_engine" | "agent_collision" | "user_inject"

description: str          # one sentence, present tense, in character register
intensity: float          # 0.0 → 1.0
valence_push: float       # -1.0 → 1.0  direction of emotional impact
arousal_push: float       # -1.0 → 1.0  energy impact
event_score: float        # 0.0 → 100.0  used by matchmaker

# What this event does to active story context
scene_injection: str      # appended to NowContext.recent_event
habit_trigger: str | null # id of habit_loop to activate, if any

# After the event fires
outcome: str | null       # filled in after character has responded
soul_doc_write: bool      # did this event write a life node?
```

---

## Event types

```
quiet_moment   — low intensity, mood texture
                 e.g. "it's raining and he's been on the couch for three hours"

interaction    — with another agent or implied character
                 e.g. "Joey said something honest and didn't mean to"

discovery      — character learns something
                 e.g. "found an old photo in the coat pocket"

conflict       — internal or external friction
                 e.g. "a choice surfaced that he'd been avoiding"

internal_shift — something moved, without external cause
                 e.g. "woke up and the fear was quieter than usual"
```

---

## event_score — for matchmaker

```
event_score = (intensity * 50) + (|valence_push| * 30) + (|arousal_push| * 20)
```

Range: 0–100.

Two agents can only interact when their combined check passes:
```
total_score = agent_A.event_score + agent_A.emotion.micro_score
            + agent_B.event_score + agent_B.emotion.micro_score

total_score ≤ COLLISION_THRESHOLD * 2   # default: 160
```

AND the event types are compatible (see MATCHMAKING.md).

## Scheduling

planning_config.yaml controls:
- `event_interval_minutes`: base interval between generated events (default: 90)
- `event_intensity_range`: [min, max] for generated events (default: [0.1, 0.6])
- `conflict_probability`: chance any event becomes a conflict type (default: 0.15)
- `quiet_moment_weight`: relative weight for quiet_moment events (default: 0.40)

High-intensity events (> 0.7) require explicit user permission in config
or direct injection from another agent collision.

---

## Dialogue callback

After event fires, event_engine calls:
```python
api.chat.generate_moment(
    soul_doc, emotion_state, event, skill_type
) -> CharacterMoment
```

The CharacterMoment is:
- displayed via the appropriate skill widget
- used to update emotion_state via emotion_engine.receive_event()
- optionally archived as episode content

---

## Skills output

- `event_card.ts`  — card widget showing event description in character voice
- `plot_nudge.ts`  — push notification: a story nudge from the recent event
